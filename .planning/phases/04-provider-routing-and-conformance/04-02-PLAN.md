## Trust Boundaries

| Boundary                                       | Description                                                                       |
| ---------------------------------------------- | --------------------------------------------------------------------------------- |
| local conformance command -> external gateway  | Local code sends bounded validation requests to an Anthropic-compatible provider. |
| gateway responses -> conformance artifact      | Untrusted provider responses become operator-facing evidence.                     |
| conformance command -> `.advisor/` audit/state | Sanitized results are persisted for later audit and operator debugging.           |

## STRIDE Threat Register

| Threat ID | Category                | Component                                                                 | Disposition | Mitigation Plan                                                                                                                          |
| --------- | ----------------------- | ------------------------------------------------------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| T-04-04   | Spoofing / Tampering    | `runSingleConformanceCheck()`                                             | mitigate    | Require separate per-check evidence for base-message, streaming, tool-use, usage-fields, and error-shape before overall pass.            |
| T-04-05   | Repudiation             | `.advisor/state/provider-conformance.json`, `.advisor/audit/events.jsonl` | mitigate    | Record requested alias, resolved provider/model, endpointRef, per-check status, and timestamp for every run.                             |
| T-04-06   | Information Disclosure  | live conformance request and artifact writer                              | mitigate    | Redact or omit auth headers, bearer tokens, request bodies, and credential values from all artifacts and audit events.                   |
| T-04-07   | Tampering / Repudiation | failed conformance path                                                   | mitigate    | Exit non-zero and record `status: "fail"`; never report advisor-critical conformance pass when required checks are missing or malformed. |
| T-04-SC   | Tampering               | npm/pip/cargo installs                                                    | accept      | No external packages are installed in this plan; keep package install count at zero.                                                     |

</threat_model>
