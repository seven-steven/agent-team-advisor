## Trust Boundaries

| Boundary                                          | Description                                                                                 |
| ------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| provider response metadata -> route audit helpers | Untrusted provider metadata is normalized into audit fields.                                |
| hook input -> .advisor artifacts                  | Runtime hook events can include provider response metadata and secret-bearing raw payloads. |
| route artifacts -> operator inspection            | Local state/audit files are used to debug provider/model routing decisions.                 |
| runtime source availability -> phase closure      | Missing Claude Code/provider metadata must stop closure rather than produce false evidence. |

## STRIDE Threat Register

| Threat ID  | Category               | Component                             | Disposition | Mitigation Plan                                                                                                                         |
| ---------- | ---------------------- | ------------------------------------- | ----------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| T-04-04-01 | Information Disclosure | normalizeServedRoute / audit builders | mitigate    | Allowlist servedRoute fields and omit headers, Authorization, messages, content, prompt, completion, and raw response bodies.           |
| T-04-04-02 | Repudiation            | advisor/executor audit artifacts      | mitigate    | Store configuredRoute/resolvedRoute separately from observed servedRoute so operators can prove whether provider-observed data existed. |
| T-04-04-03 | Tampering              | served-route mismatch handling        | mitigate    | Preserve mismatches as evidence instead of overwriting configuredRoute or silently normalizing provider output to static config.        |
| T-04-04-04 | Spoofing               | ROUT-02 closure source validation     | mitigate    | assertServedRouteSourceAvailable rejects configured/resolved route data as actual served-route source-of-truth.                         |
| T-04-04-SC | Tampering              | npm/pip/cargo installs                | accept      | No package installs are planned; implementation uses existing CommonJS modules and Node built-ins.                                      |
