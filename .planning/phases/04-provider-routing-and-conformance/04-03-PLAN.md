<threat_model>

## Trust Boundaries

| Boundary                               | Description                                                                         |
| -------------------------------------- | ----------------------------------------------------------------------------------- |
| operator env -> conformance CLI        | Secrets and gateway endpoints enter local runtime through environment variables.    |
| conformance CLI -> external gateway    | Bounded live HTTP requests leave the repo and receive untrusted provider responses. |
| gateway response -> .advisor artifacts | Provider response metadata becomes local state/audit evidence.                      |

## STRIDE Threat Register

| Threat ID  | Category               | Component                                 | Disposition | Mitigation Plan                                                                                                                                |
| ---------- | ---------------------- | ----------------------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| T-04-03-01 | Information Disclosure | createLiveGatewayClient / artifact writer | mitigate    | Read ANTHROPIC_AUTH_TOKEN only from env/options, omit Authorization headers and token values from stdout, state JSON, audit JSONL, and errors. |
| T-04-03-02 | Spoofing               | live gateway response parsing             | mitigate    | Require Anthropic-compatible response structure per check and mark malformed responses fail instead of pass.                                   |
| T-04-03-03 | Tampering              | configuredRoute vs servedRoute evidence   | mitigate    | Store configured route and live response served route separately so static config cannot masquerade as provider-observed output.               |
| T-04-03-SC | Tampering              | npm/pip/cargo installs                    | accept      | No package installs are planned; implementation uses Node built-ins and global fetch.                                                          |

</threat_model>
