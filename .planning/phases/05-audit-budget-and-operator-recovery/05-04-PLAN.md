| --------------------------------------- | ------------------------------------------------------------------ |
| project files -> doctor report          | Local config and agent files become operator-facing health status. |
| environment -> doctor output            | Env vars may be present while doctor runs.                         |
| doctor runtime artifacts -> audit/state | Doctor writes operational evidence to local runtime files.         |

## STRIDE Threat Register

| Threat ID | Category               | Component             | Disposition | Mitigation Plan                                                                                                                   |
| --------- | ---------------------- | --------------------- | ----------- | --------------------------------------------------------------------------------------------------------------------------------- |

| T-05-11   | Tampering              | doctor command        | mitigate    | Doctor is read-only for project assets; tests compare protected file contents before and after execution.                         |
| T-05-12   | Spoofing               | provider route health | mitigate    | Doctor separates configured route checks from observed provider conformance state and does not fabricate observed models.         |
| T-05-SC   | Tampering              | package installs      | accept      | Phase 5 installs no packages; package legitimacy audit lists none.                                                                |

</threat_model>
