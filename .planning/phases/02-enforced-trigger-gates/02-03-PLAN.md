Disposition fields: `correlationKey`, `event: "human_approval.disposition"`, `disposition`, `decidedBy`, `decidedAt`, `rationale`, `appliesTo`, and optional `conditions` for revise/defer.
A valid disposition changes local re-entry state to `satisfied` only when its `correlationKey` and `appliesTo.event` match the blocked packet.
AskUserQuestion is the expected workflow-layer surface for collecting the human decision from the packet, but the hook itself does not await the answer; execution resumes only after a disposition artifact exists and the user or executor explicitly retries the blocked action.
