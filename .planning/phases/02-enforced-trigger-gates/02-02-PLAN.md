Host contract: high-risk first attempts must use supported PreToolUse fields to deny the current invocation and provide retry guidance. Re-entry is an explicit retry after the recommendation artifact exists and validates; only then may the hook allow the action.

For identifiable configured gate matcher events (`Bash|Edit|Write|MultiEdit`) that would otherwise enter policy evaluation, policy-load failure, classification failure, and required consultation request artifact write failure are hard-stop conditions. Represent these as explicit blocking failures using supported PreToolUse semantics and/or exit code 2; do not rely on unsupported custom host metadata.
